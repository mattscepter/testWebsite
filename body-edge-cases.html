Edge-case fixtures: documents with no <body> element

These pages exercise the "skip pages without a body tag" fix. In each, document.querySelector('body') is null (or the body is unavailable at scan time), which previously caused the Cannot read properties of null (reading 'outerHTML') crash.

  • nobody.html - HTML with no <body> in markup
  • frameset.html - frameset document (no body element)
  • data.xml - plain XML document
  • feed.xml - RSS feed (XML)
  • image.svg - standalone SVG (XML)
  • nobody.xhtml - XHTML (application/xhtml+xml) with no body
  • slow-body.html - render-blocking head; body parsed late

Note: the XML/SVG/XHTML cases must be served with their correct content type (via GitHub Pages, not raw.githubusercontent.com) for the parser to skip body synthesis.
